## General

Please `contributtors` should make sure that they've gone through, and are ready to comply with, the `GPL-3.0 license` terms and conditions, as this project is distributed under such terms.

Contributors have to `fork` this repo into their account, then any changes made should be pushed to this forked repo, whereafter a `pull request` is made with a reference to the forked repo.

Note that all your codes should, and must strictly follow the betty coding and documentation style, and no more than five functions in a file. Anything contrary to that, your pull request will only be taken note of, and the pull request will surely receive comment on why it was ignored. Merges will be made omly after these criteria has been met. This is so in order to create clean, accepteble code, to enhance readability for other contributors.

## Creating pull requests

Pull requests should be made giving a clear explanation of what has changed.

## Creating issues

Please `issues` should make use of tags when possible. For instance, an issue relating to a bug should use the `bug` tag.
